## 5. Benchmark definition (must satisfy)

A Polar Lab benchmark is valid only if:

1. **Disjoint** train vs holdout prompts  
2. **Holdout never trained on**  
3. **Automatic gold** + frozen decode (`temperature=0`)  
4. Metric = **`exact_match`** (case-insensitive strip)  
5. Always report **base vs LoRA**  
6. Target behavior matches the data domain (here: Machina/Centaur facts)

### Pass bars

| Level | Criteria |
|-------|----------|
| Smoke | Job completes; metrics file written |
| Useful (GPU target) | Holdout `exact_match` **≥ 0.60** on short-fact set |
| Clear win | LoRA **≥ base + 0.20** |
| Invalid claim | Chat-only vibes; leaked eval into train |

CPU baseline to beat on GPU: **0.200**.

## 6. How to run for real on GPU

Full copy-paste: **[docs/GPU_RUNBOOK.md](../docs/GPU_RUNBOOK.md)**

Short path:

```bash
nvidia-smi
git clone https://github.com/lilaclilac09/polar-lab.git && cd polar-lab
python3 -m venv .venv && source .venv/bin/activate
pip install torch --index-url https://download.pytorch.org/whl/cu124
pip install -r requirements.txt

python scripts/check_data.py
python scripts/01_sft.py --config configs/machina_sft.yaml   # device must be cuda
python scripts/05_eval_holdout.py --adapter outputs/sft/adapter
python scripts/05_eval_holdout.py --adapter "" --metrics-out outputs/eval/metrics_base.json
```

Log GPU name, steps, losses, and both exact_match numbers into `logs/week_01.md`.
